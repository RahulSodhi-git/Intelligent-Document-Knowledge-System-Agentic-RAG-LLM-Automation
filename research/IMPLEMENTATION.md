# RIM Implementation — What Was Built (July 2026)

All 5 phases from UPGRADE_PLAN.md are now implemented. Summary of every change:

## New files

**`intelligence/memory.py`** — the RIM core (episodic memory).
Every build attempt (canonical, adhoc, live) is written to `output/experience.json` as one
experience: task, outcome, verify score, gate decision, and a *reflection* explaining why —
generated by the model but grounded strictly in the verify report (heuristic summary without
a model). Failures are stored too. `retrieve(task)` finds the most similar past experiences
(dependency-free token matching); `context_for(task)` builds the prompt block agents read
before acting. Also holds lasting chat facts in `output/chat_memory.json` with Mem0-style
ADD/UPDATE/DELETE consolidation. Diary capped at 400 entries.

**`intelligence/distill.py`** — lessons distillation.
Reads high-confidence experiences and rewrites `output/lessons.md` (max ~30 one-line rules,
scoped like `[Self-service]`). With a model: full consolidation (keep/update/delete rules
against evidence). Without: a heuristic counts recurring verified problems (threshold: seen
2+ times) and regenerates from scratch — built-in decay. Run: `python -m intelligence.distill`
or `POST /api/distill`.

## Changed files

**`intelligence/verify.py`** — reports now carry `high_confidence` (vision saw the frames, or
only deterministic checks fired). Only high-confidence outcomes may teach lessons — protects
memory from judge errors.

**`intelligence/resolver.py`** — selector QUARANTINE: learned selectors need 2+ recorded
successes before they outrank the hand-written library; one lucky hit stays provisional at
the back of the queue. New `record_heal()` logs runtime heals to `output/heals.json`, the
resolver cache (in quarantine), and the experience diary. `stats()` now reports heals.

**`capture/engine.py`** — self-healing clicks: when every recipe selector fails (UI drift),
`_heal_click()` asks the model to pick the intended control from the page's real elements
(`_CANDS_JS`), clicks its true geometry, and records the healed selector in quarantine.
Safety: healed clicks are refused on destructive controls (delete/pay/confirm/etc.).

**`intelligence/live_agent.py`** — before driving a new task, the agent loads
`memory.context_for(task)` (similar past experiences + distilled lessons) into every
decision prompt. It starts from experience instead of zero.

**`server.py`** —
- `_run_build` and `_run_live_build` record every attempt into the diary after verify/gate;
- chat now injects remembered user facts into all answer tiers and extracts new lasting
  facts in a background thread (zero added latency);
- new endpoints: `GET /api/memory/status`, `POST /api/distill`.

**`intelligence/orchestrator.py`** — `health()` now includes memory + lessons stats.

**`intelligence/__init__.py` / `intelligence/README.md`** — documentation of the new loop:
SEE → PLAN → CAPTURE → BUILD → VERIFY → REMEMBER → DISTILL → LEARN.

## Research → code mapping

| Research finding | Where it lives |
|---|---|
| Store reasoning from successes AND failures (ReasoningBank) | `memory.record_build` stores every outcome |
| Retrieve experiences before acting (ExpeL/AWM) | `memory.context_for` → `live_agent` prompts |
| Distill episodes into insights (ExpeL/Generative Agents reflection) | `distill.run` → `lessons.md` |
| Self-critique needs external grounding (Huang et al.) | reflections built ONLY from verify reports; `high_confidence` gate |
| Consolidation + forgetting beats accumulation (Mem0/MemoryBank) | capped diary, rule cap + decay, chat-fact ADD/UPDATE/DELETE |
| Online-learned workflows need quality control (AWM) | selector quarantine (2 wins to trust) |
| Rank real DOM candidates, not marks (SeeAct) | `_CANDS_JS` + `pick_element` healing path |
| DOM-diff selector repair (Healenium pattern) | `_heal_click` + `record_heal` + `heals.json` |

## How to activate the full system

1. In `config/intelligence.json`, change `"provider": "none"` to `"provider": "openai"`
   (a key is already present). Everything degrades gracefully without it.
2. Build/rebuild questions as usual — every attempt now feeds the diary automatically.
3. After a batch of builds, run `python -m intelligence.distill` (or POST /api/distill).
4. Watch it learn: `python -m intelligence.orchestrator health` or `GET /api/memory/status`.

## Success metrics to track

Average verify score across the 44 questions; heals logged vs builds aborted
(`output/heals.json`); live-agent steps per task; 👍 rate in `output/feedback.json`;
rules in `output/lessons.md` that survive multiple distills.
